Сервер имеет 5 методов:

1) Index.
	Почти ничего не делающий метод. Выводит надпись "Users REST API".
	Метод: GET.
	Запрос: "/".

2) Get user.
	Выдает имя пользователя из БД по Id.
	Метод: GET.
	Запрос: "/users/{id}".
	Ответ: {"meta":{"code":<http_код_результата>},"response":{"id":<айди_пользователя>,"name":<имя_пользователя>}}

3) Create user.
	Добавляет пользователя в БД. Id пользователю выдается равным (Id_последнего_добавленного_пользователя + 1).
	Метод: POST.
	Headers: Content-Type = application/json.
	Запрос: "/users/create".
	Тело запроса: {"name":"имя_нового_пользователя"}. Поле "name" должно присутствовать. Остальные поля игнорируются.
	Ответ: {"meta":{"code":<http_код_результата>},"response":<айди_пользователя>}

4) Update user.
	Меняет имя существующего в БД пользователя.
	Метод: POST.
	Headers: Content-Type = application/json.
	Запрос: "/users/{id}/update".
	Тело запроса: {"name":<новое_имя_пользователя>}. Поле "name" должно присутствовать. Остальные поля игнорируются.
	Ответ: {"meta":{"code":<http_код_результата>}}

5) Delete user.
	Меняет имя существующего в БД пользователя.
	Метод: POST.
	Headers: Content-Type = application/json.
	Запрос: "/users/{id}/delete".
	Ответ: {"meta":{"code":<http_код_результата>}}
